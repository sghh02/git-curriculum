# 04 PRの作成とセルフレビュー

## 1. この章のゴール
- PR の作成方法と書くべき内容を説明できる。
- セルフレビューの観点を理解する。
- PR を挟む意味を言語化できる。

## 2. 概念の説明（図や比喩を使ってわかりやすく）
- PR は「変更点の説明付き提出書」。
- セルフレビューは「提出前の読み直し」。見落としを減らし、他者の理解も助けます。

## 3. 実際の操作手順（git コマンド）
1. feature ブランチを push する。
   ```bash
   git checkout -b feature/submit-task
   git push -u origin feature/submit-task
   ```
2. GitHub UI で PR を作成する。
   - Base: `develop`
   - Compare: `feature/submit-task`
3. PR テンプレート例:
   - 変更内容
   - 背景 / 目的
   - 確認方法
   - 影響範囲

## 4. よくあるミス・つまずきポイント
- PR の説明が短すぎて意図が伝わらない。
- レビュー前にローカルでテストしていない。
- Base ブランチを main にしてしまう。

## 5. AIに聞いてみよう（質問例）
- 「良い PR 説明文の例を作って」
- 「セルフレビューで確認すべき観点をチェックリスト化して」
- 「PR を挟むことで得られるメリットは？」

## 6. ハンズオン課題（成果物提出を想定）
- `feature/submit-task` ブランチで課題ファイルを追加する。
- PR を作成し、説明文に「目的・変更点・確認方法」を記載する。
