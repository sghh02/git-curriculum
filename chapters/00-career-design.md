# キャリア設計と学習設計

> **この章は、技術に入る前の接続フェーズです。**
> 前のAIカリキュラムで作成した最終成果物を、このGitカリキュラムの題材として再利用します。

## 1. この章のゴール

- 前のAIカリキュラムで作った「自分のエンジニア設計書」を、このGitカリキュラムに接続する
- 学習の二層構造（思考→技術）を確認する
- AI活用の原則（整理→質問→検証）を、このカリキュラムでも同じく使う
- Git を「ただの提出手段」ではなく、「思考の改善履歴を残す道具」として捉える
- 次章で「Git とは何か」を理解するための前提をそろえる

## 2. やること（5〜10分）

### 2-1. 前カリキュラムの最終成果物を確認する

前のAIカリキュラム「AIと問いから始める学び方」で作成した、**自分のエンジニア設計書**を開いてください。

このGitカリキュラムでは、その内容を毎回少しずつ改善していきます。

見るポイントは次の3つです。

- なぜエンジニアを目指すのか
- どんなエンジニアになりたいのか
- 最初の3か月で何をするつもりか

> まだ書き直さなくて大丈夫です。まずは「この内容を Git で育てていく」と理解できればOKです。

この章では、まだ Git の操作はしません。  
先に「何を題材にするのか」「なぜそれを Git で扱うのか」を固定しておくのが目的です。

## 3. 学習の二層構造

このカリキュラムは、次の順で進みます。

1. **第一層（思考）**：問いの設計、AIとの協働、自己管理
2. **第二層（技術）**：Git/GitHubの手順、提出ワークフロー

この章では第一層の内容を「再利用」し、次章から第二層に入ります。  
つまり、このGitカリキュラムで扱う提出物は「技術練習のためのダミー」ではなく、**自分の将来設計そのもの**です。

## 4. AI活用の原則（このカリキュラム共通）

AIは「答えをもらう箱」ではなく、**考える相手**として使います。

- 先に自分で状況を整理する
- 仮説を持って質問する
- 返答は必ず検証する

### 質問テンプレ

```md
目的：
現状：
詰まりポイント：
自分の仮説：
確認したいこと：
```
