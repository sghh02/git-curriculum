# 00 AIと一緒に学ぶGit/GitHub

## 1. この章のゴール
- Git / GitHub の学習で AI を「理解の補助輪」として使う方法を知る。
- 以降の章で「なぜその運用が良いのか」を説明できる準備をする。

## 2. 概念の説明（図や比喩を使ってわかりやすく）
- Git を「作業の履歴ノート」、GitHub を「履歴ノートをみんなで置く共有棚」と考えます。
- AI は「いつでも質問できる先輩」です。先輩に理由を説明させることで理解が深まります。

## 3. 実際の操作手順（git コマンド）
1. まずは学習用ディレクトリを用意します。
   ```bash
   mkdir git-learning
   cd git-learning
   ```
2. 以降の章では、このフォルダで Git の操作を行います。

## 4. よくあるミス・つまずきポイント
- AI に質問する前に「何がわからないのか」を言語化しない。
- コマンドの意味を確認せずにコピペだけで進めてしまう。

## 5. AIに聞いてみよう（質問例）
- 「Git の commit は、なぜ“スナップショット”と説明されるの？」
- 「GitHub があると、Git だけではできないことは？」
- 「ブランチを切る理由を、初心者向けに例え話で説明して」

## 6. ハンズオン課題（成果物提出を想定）
- 学習用の「疑問ノート」を作成し、AI に聞いた内容を追記します。
  1. `notes.md` を作成する。
  2. AI に質問し、回答を箇条書きで保存する。
  3. 次章で Git で管理する準備をする。

- [ ] AI に質問する目的を自分の言葉で説明できる。
- [ ] Git と GitHub の違いを比喩で説明できる。
- [ ] 学習用フォルダとメモの準備ができた。
