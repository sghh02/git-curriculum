# 02 add/commit/pushとログの読み方

## 1. この章のゴール
- `add` / `commit` / `push` の役割を説明できる。
- `status` / `diff` の読み方を理解する。
- こまめに commit する理由を言語化できる。

## 2. 概念の説明（図や比喩を使ってわかりやすく）
- `add` は「提出物の下書きを提出トレーに乗せる」操作。
- `commit` は「提出トレーの内容を封筒に入れて封をする」操作。
- `push` は「封筒を郵送して共有棚に置く」操作。
- こまめな commit は「作業の里程標」を残し、戻れる地点を増やします。

## 3. 実際の操作手順（git コマンド）
1. 状態を確認する。
   ```bash
   git status
   ```
2. 変更点を確認する。
   ```bash
   git diff
   ```
3. 変更をステージする。
   ```bash
   git add README.md
   ```
4. コミットする。
   ```bash
   git commit -m "Add repository overview"
   ```
5. リモートに送る。
   ```bash
   git push origin main
   ```

## 4. よくあるミス・つまずきポイント
- まとめて巨大な commit になり、差分が追えない。
- `git add .` の前に `git diff` を見ずに不用意なファイルを含めてしまう。
- メッセージが「update」だけで内容が分からない。
